# 日本株 短期複合戦略バックテスト

地合いレジーム判定で「順張りスイング」と「逆張りスイング」を切り替え、決算シーズンは「決算PEAD」を並走させる複合戦略のバックテスト環境。株価・決算・信用残データは J-Quants API V2 から取得する。

## 戦略の構成

| モジュール | 稼働条件 | 概要 |
|---|---|---|
| ① レジーム判定 | 毎日 | TOPIXが25日線上+傾き上向きなら順張り、それ以外は逆張りを稼働。ADX20未満はサイズ半分 |
| ② 順張りスイング | 上昇レジーム | RS上位10%銘柄の3〜7日押し目(-5〜-12%)を反転陽線で買い。2Rで半分利確、10日線割れで撤退 |
| ③ 逆張りスイング | レンジ・下落レジーム | 25日線乖離-15%以下+RSI(2)<10+出来高2倍で買い。25日線への半値戻しで利確、-7%損切り |
| ④ 決算PEAD | 常時 | 決算翌日+5%以上ギャップ+高値引けの銘柄を、2〜5日後の押し目で買い。窓埋めで損切り、20日で手仕舞い |
| ⑤ 信用需給フィルター | 全エントリー | 信用倍率5倍超の銘柄は新規買い見送り(公表ラグ3営業日を考慮) |
| ⑥ カレンダーフィルター | 全ポジション | メジャーSQ週はサイズ半分。決算発表2営業日前(PEADは5営業日前)までに手仕舞い |
| ⑦ 仕手株・低流動性除外 | 全エントリー | ユニバースは売買代金の中央値で選定(スパイク耐性)。日々公表(信用規制)掲載銘柄は10営業日エントリー禁止。逆張りは300円未満の低位株とTOPIX Mid400未満の小型株を除外。建玉は20日平均出来高の5%が上限 |

資金管理: 1トレードのリスク=資産の1%、最大5銘柄、月間DD-5%で当月の新規建て停止。

## セットアップ

```powershell
.venv\Scripts\activate            # 仮想環境(作成済み)
copy .env.example .env            # 認証情報を記入
```

`.env` に J-Quants の APIキーを設定する([J-Quantsダッシュボード](https://jpx-jquants.com/)から発行)。

本プロジェクトは **API V2** 仕様(`https://api.jquants.com/v2`、`x-api-key` ヘッダー認証)。V1のリフレッシュトークン方式は使わない。2025年12月22日以降の新規登録アカウントはV2のみ利用可能。

## 実行

```powershell
python main.py selftest           # API不要。合成データでエンジンを検証
python main.py run                # データ取得+バックテスト(初回は取得に時間がかかる)
python main.py run --start 2023-01-01 --end 2026-03-31 --universe 300
```

結果は `output/` に出力される: `summary.txt`(成績サマリー)、`trades.csv`(全トレード明細)、`equity_curve.png`(資産曲線)。

APIレスポンスは `data_cache/` にキャッシュされ、2回目以降はAPIを叩かない。期間や銘柄数を変えて再取得したい場合は `data_cache/` を削除する。

## J-Quants プランと機能の対応

| プラン | 使える機能 | 制約 |
|---|---|---|
| Free | ②③④が動作(地合いは代替指数) | 株価が12週遅延・過去2年分のみ。TOPIX・信用残なし |
| Light | + TOPIXで正確な地合い判定 | 過去5年分。信用残なし(⑤が無効) |
| Standard 以上 | + 信用需給フィルター⑤が有効 | 過去10年分。フル機能で動く推奨プラン |

プラン外のエンドポイントは自動検出して該当機能だけ無効化するので、どのプランでも動く。

## 既知の制約(結果を解釈する際の注意)

- ユニバースは「現在の」売買代金上位で選ぶため生存バイアスがある。厳密には各時点で再構成が必要
- 決算またぎ回避は実際の開示日を「事前に分かっていた」ものとして扱う(実運用では発表予定日を使うため概ね妥当)
- PEADのサプライズ判定は株価反応(ギャップ+高値引け)ベースの簡易版。業績予想の修正率は使っていない
- ストップ高・ストップ安の張り付き(約定不能)はモデル化していない
- パラメータは config.py の初期値のままで、最適化はしていない
